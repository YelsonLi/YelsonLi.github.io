一.

只有commit过的分支使用 git branch -a 等查询分支命令才能看到分支名

比如要修改分支名，先commit再git branch -m “xxx”

二.

从Github/Gitee创建的远程仓库项目分支fetch到本地后，idea会自动创建红色部分文件：

git fetch并没有建立upstream，所以不可使用简化命令

.gitignore只能忽略那些原来没有被track的文件，如果某些文件已经被纳入了版本管理中，则修改.gitignore是无效的，解决方法：

我们需要使用’git rm -r –cached .’命令清除一下相关的缓存内容.这样文件将以未追踪的形式出现.然后再重新添加提交一下,.gitignore文件里的规则就可以起作用了
